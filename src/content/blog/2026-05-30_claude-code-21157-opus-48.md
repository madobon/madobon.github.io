# はじめに

Claude Code がバージョン 2.1.157 へ更新された。
Opus 4.8 のサポート強化と動的ワークフロー機能が追加された。
プラグイン管理や agents 機能の安定性も向上している。

# 変更点の概要

- Opus 4.8 のデフォルト設定変更と Fast Mode 導入
- 動的ワークフローによる複数エージェント orchestration
- `.claude/skills` ディレクトリでのプラグイン自動読み込み
- `claude agents` 機能の拡張と安定性修正
- 多数の UI・UX 改善とバグ修正

# 詳細解説

## モデルと Effort 設定

Opus 4.8 がデフォルトで high effort 設定となった。
 hardest タスクには `/effort xhigh` が推奨される。
Opus 4.8 の Fast Mode が利用可能になった。
標準レートの 2 倍で 2.5 倍の速度を実現する。
 `/effort` スライダーのラベルが変更された。
「Speed」「Intelligence」から「Faster」「Smarter」へ変更された。
 lean system prompt がデフォルトになった。
Haiku、Sonnet、Opus 4.7 以前を除く全モデルで適用される。
複数選択質問プロンプトの使用頻度が減った。
コンテキストが十分な場合、Claude が自ら判断する。
`CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE` は非推奨となった。
2026 年 6 月 1 日に削除される。
Opus 4.6 で Fast Mode を使う場合はモデル切り替えが必要だ。
 `/model claude-opus-4-6[1m]` 後に `/fast on` を実行する。
VSCode の Fast Mode インジケーターが Opus 4.8 で表示されるようになった。

## 動的ワークフローと Agents

動的ワークフロー機能が導入された。
Claude にワークフロー作成を依頼すると背景で orchestration する。
数十から数百の agents が連携して複雑なタスクを処理する。
 `/workflows` コマンドで実行履歴を確認できる。
`claude agents` で `! <command>` が入力可能になった。
シェルコマンドを背景セッションとして実行できる。
アタッチやデタッチも可能だ。
`claude --bg --exec '<command>'` でも同等の操作ができる。
`settings.json` の `agent` フィールドが dispatch セッションで尊重される。
`--agent <name>` でオーバーライド可能だ。
背景セッションの分類器がユーザーの目標を保持するようになった。
スケジュールされた `/command` 発火時にも維持される。
固定された背景セッションの再spawn ループが修正された。
アップデート後のアイドル時にプロセス churn が発生しない。
背景セッションが「blocked」等で停滞しても retirement する。
サブエージェントが worktree 隔離ガードをバイパスする問題が修正された。
共有チェックアウトへの書き込みが防止される。
`/logout` が背景セッション送信ではなくサインアウトする。
`←←` で agents ビューが開けるようになった。
Bedrock、Vertex、Foundry、 telemetry 無効時でも利用可能だ。
IDE で Stop クリック時に背景サブエージェントが停止するようになった。
`--resume` が終了時の背景サブエージェントを報告する。
`--resume` セッションピッカーがフルスクリーン後に残留しない。
agents の dispatch 入力で slash-command 補完が部分一致する。
agents の完了セッションがアイドルサブエージェント時に retirement する。
agents で Esc 押下時に「opening…」のキャンセルが効くようになった。
背景エージェントの完了通知が 1M コンテキストモデルで誤動作しない。

## プラグインと Skills

 `.claude/skills` ディレクトリのプラグインが自動読み込みされる。
マーケットプレイス不要で利用可能だ。
`claude plugin init <name>` で新規プラグインを scafolding できる。
 `/plugin` 引数の autocomplete が強化された。
サブコマンド、インストール済みプラグイン、既知マーケットプレイスが表示される。
プラグインで `defaultEnabled: false` を宣言可能になった。
`plugin.json` またはマーケットプレイスエントリーで設定する。
 `/plugin` または `claude plugin enable` で有効化する。
依存プラグインは自動で有効化される。
 `/plugin` Discover タブでディレクトリ適合プラグインがピン留めされる。
「suggested for this directory」注釈が表示される。
 `/claude-api` スキルに Opus 4.8 サポートが追加された。
4.7 から 4.8 への移行ガイダンスも含まれる。

## Worktrees と環境

`EnterWorktree` でセッション中に worktree 切り替えが可能になった。
Claude 管理の worktrees はエージェント終了後にロック解除される。
`git worktree remove` や `prune` でクリーンアップ可能だ。
背景エージェントの worktrees が 30 日保持スイープ後に orphan 化しない。
`--worktree` と `--worktree --tmux` が現在の linked worktree に戻る。
 canonical repo root ではなくリンク先を維持する。
`worktree.baseRef: "head"` が現在の worktree の HEAD を解決する。
メインチェックアウトの HEAD ではなく参照される。
`$TMPDIR` がサンドボックス内外で異なるディレクトリを指す問題が修正された。
同一セッション内で整合性が保たれる。
サンドボックスネットワーク権限プロンプトが表示されないようになった。
auto および bypass-permissions モードでのデスクトップアプリ等での問題だ。
Stdio MCP サーバーサブプロセスに環境変数が渡される。
`CLAUDE_CODE_SESSION_ID` と `CLAUDECODE=1` が追加された。
`claude mcp list` と `get` で未承認サーバーが表示される。
`⏸ Pending approval` として表示され自動承認されない。
`/remote-control` autocomplete に disconnect 选项が追加された。
Remote Control  active 時に表示される。

## UI・UX とターミナル

ストリーミングツール実行が常に有効化された。
telemetry 無効時や Bedrock、Vertex、Foundry でも利用可能だ。
以前は機能フラグ背後にあった。
WSL で画像ペーストが修正された。
`alt+v` キーバインドと Windows 11 のスクリーンショットペーストだ。
Windows Explorer からの画像ドラッグもサポートされた。
長時間会話と再開会話のパフォーマンスが向上した。
冗長なメッセージレンダリング再計算が削除された。
 `/terminal-setup` が GPU 加速を無効化する。
VSCode、Cursor、Windsurf 統合ターミナルでの文字化け防止だ。
Feature of the Week クレジット主張ステータスが通知エリアに表示される。
プロンプト上の行ではなくステータスエリアだ。
「bash commands will be sandboxed」スタートアップバナーが削除された。
サンドボックスステータスは `/status` とコマンドブロック時に表示される。
「/ide for …」スタートアップヒントトーストが削除された。
ワークフローキーワードトリガー設定が `/config` に追加された。
プロンプト内の「workflow」単語で動的ワークフローが発動しない設定だ。
ワークフロー発動キーワード直後の backspace でリクエストが解除される。
`alt+w` と同様の挙動だ。
コピーオンセレクトが tmux 内でシステムクリップボードに到達する。
`set-clipboard on` 設定時の 2.1.153 での回帰問題が修正された。
ターミナル UI 再描画後のスクロールバック重複行が修正された。
VSCode、Cursor、Windsurf 統合ターミナルの右クリックペースト重複が修正された。
スタートアップ時の managed-settings セキュリティダイアログ承認後のフリーズが修正された。
フルスクリーンモードの進行中メッセージに markdown マーカーが表示されない。
 `/model` ピッカーのヒント表示が修正された。
選択モデルが最新の場合の「Newer version available」ヒントだ。
ピン留めモデル行に raw ID ではなくモデル説明が表示される。
オプションダイアログの区切り線以下の番号キーショートカットが機能する。
ハイライトされた行のテキストが読みやすくなった。
Claude Code テーマがターミナル背景と一致しない場合の問題だ。
画像ペースト時のクラッシュが修正された。
ゼロバイトや破損画像がテキストプレースホルダーになる。
`rm -rf $HOME` が危険パスとしてブロックされる。
`HOME` に末尾スラッシュがある場合も対象だ。
自動モード分類器のデータ流出検出が改善された。
リポジトリ内容の bulk transfers 検出だ。
背景セッションがスリープ/ウェイク後に正しい日付をモデルに伝える。
Orphan 化した `claude --bg-pty-host` プロセスの CPU 使用率が修正された。
macOS でデーモン終了後の 100% CPU スピン問題だ。
Opus 4.8 使用時の thinking blocks 変更による API エラーが修正された。

# まとめ

このアップデートで大規模タスク処理能力が大幅に向上した。
動的ワークフローにより複数エージェント連携が可能になった。
Opus 4.8 の Fast Mode でコストパフォーマンスが改善される。
プラグイン管理の簡素化と agents 機能の安定化もメリットだ。
開発者はより複雑な作業を効率的に自動化できる。
