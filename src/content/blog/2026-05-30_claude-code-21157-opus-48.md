## はじめに

Claude Code がバージョン 2.1.157 へ更新された。
Opus 4.8 の正式サポートや動的ワークフロー機能が追加された。
プラグインシステムの拡張により、拡張性が大幅に向上している。

## 変更点の概要

主要な変更点は以下の通りだ。

- Opus 4.8 モデルのデフォルト採用と Fast Mode 強化
- 複数エージェントを orchestrate する動的ワークフロー
- .claude/skills ディレクトリによるプラグイン自動読み込み
- agents 機能の安定性向上と設定項目の追加
- 多数のバグ修正と UI/UX の改善

## 詳細解説

### モデルとパフォーマンス

Opus 4.8 が正式にリリースされた。
デフォルトで high effort 設定となり、難易度の高いタスクに対応する。
`/effort xhigh` でさらに高い努力レベルを指定可能だ。
Opus 4.8 の Fast Mode が利用可能になった。
標準料金の 2 倍で 2.5 倍の速度を実現する。
コスト効率が previous version より改善された。
lean system prompt がデフォルトで適用される。
Haiku、Sonnet、Opus 4.7 以前を除く全モデルが対象だ。
モデル選択時のヒント表示が修正された。
最新モデル選択時に「Newer version available」が出なくなった。
 pinned-model 行に raw ID ではなく説明が表示される。
2.1.156 では Opus 4.8 の thinking blocks 修正による API エラーを修正した。

### 動的ワークフロー

動的ワークフロー機能が導入された。
Claude にワークフロー作成を依頼すると、数十から数百の agents を裏で orchestrate する。
大規模で複雑なタスクを処理可能だ。
`/workflows` コマンドで実行履歴を確認できる。
プロンプト内の「workflow」という単語でトリガーされない設定を追加した。
`/config` から「Workflow keyword trigger」設定を無効化できる。
ワークフロー触发キーワード直後の backspace 入力でリクエストを dismiss する。
alt+w キーバインドと同様の動作だ。

### プラグインシステム

`.claude/skills` ディレクトリの plugins が自動読み込みされる。
marketplace を経由せず利用可能だ。
`claude plugin init <name>` で新しい plugin を scaffold できる。
`/plugin` 引数の autocomplete が強化された。
subcommands、installed plugin names、known marketplaces の plugins が対象だ。
plugins は `plugin.json` で `defaultEnabled: false` を宣言可能だ。
`/plugin` または `claude plugin enable` で有効化する。
依存関係にある plugins は自動的に有効化される。
`/plugin` Discover タブでディレクトリに適合する plugins がピン留めされる。
「suggested for this directory」注釈が表示される。

### agents とセッション管理

`claude agents` 機能の安定性が向上した。
`settings.json` の `agent` フィールドが dispatched sessions で尊重される。
`--agent <name>` オプションでオーバーライド可能だ。
背景セッションの実行方法が追加された。
`! <command>` で背景セッションとして shell command を実行できる。
`claude --bg --exec '<command>'` でも利用可能だ。
`/logout` は背景セッションではなくサインアウト動作になった。
`←←` キーで agents view を開けるようになった。
Bedrock、Vertex、Foundry、telemetry 無効時でも動作する。
dispatch 入力での slash-command autocomplete が substring match する。
完了した sessions が idle subagent で retiring されない問題を修正した。
Esc キーで遅い「opening…」をキャンセルできるようになった。
`--resume` で前プロセス終了時の背景 subagents を報告する。
`--resume` session picker が fullscreen 終了後に内容を残さない修正した。
背景 sessions の sleep/wake 後の日付修正を行った。
tmux 内での copy-on-select が system clipboard に到達する修正した。
IDE で Stop クリック時に背景 subagent が停止しない問題を修正した。
VSCode で Opus 4.8 の fast mode indicator が表示される修正した。
背景 sessions の classifier がユーザーの goal を丢失する問題を修正した。
pinned background sessions の update 後 respawn 問題を修正した。
背景 sessions が idle grace period 後に retiring しない問題を修正した。
subagents が worktree-isolation guard を bypass する問題を修正した。
orphaned `claude --bg-pty-host` processes の CPU 消費問題を修正した。

### Worktree と Git 統合

`EnterWorktree` でセッション中に Claude-managed worktrees を切り替え可能だ。
agent 終了後に worktrees が unlock される。
`git worktree remove` や `prune` で cleanup 可能だ。
背景 agent worktrees が 30 日保持 sweep 後に orphan になる問題を修正した。
`--worktree` と `--worktree --tmux` が current linked worktree に戻る修正した。
`worktree.baseRef: "head"` が current worktree の HEAD を参照する修正した。

### UI/UX とターミナル

WSL での画像貼り付け機能が改善された。
`alt+v` キーバインドと Windows 11 の screenshot paste をサポートする。
Windows Explorer からの画像ドラッグも可能だ。
長文会話と resumed 会話の rendering 性能を改善した。
redundant message-rendering recomputations を eliminated した。
`/terminal-setup` で VSCode/Cursor/Windsurf 統合ターミナルの GPU 加速を無効化する。
garbled-text rendering を防止する。
Feature of the Week の credit-claim 状態が status area の notification で表示される。
startup banner の「bash commands will be sandboxed」を削除した。
sandbox 状態は `/status` と command blocked 時に表示される。
「/ide for …」startup hint toast を削除した。
fullscreen モードでの in-progress message text の markdown markers 表示を修正した。
startup の managed-settings security dialog 承認後の terminal freeze を修正した。
terminal UI redraw 後の scrollback 重複行問題を修正した。
VSCode、Cursor、Windsurf 統合ターミナルでの右クリック paste 重複を修正した。
option dialogs の divider 下部オプションの number key shortcuts を修正した。
背景 agent 完了通知による premature "out of context" 動作を修正した。
`claude agents` の highlighted-row text 読みやすさを改善した。

### セキュリティと MCP

`tool_decision` telemetry events に `tool_parameters` を含むようになった。
`OTEL_LOG_TOOL_DETAILS=1` 設定時に bash commands や MCP/skill names が記録される。
sandbox network permission prompts の表示問題を修正した。
auto と bypass-permissions モードでの desktop app などが対象だ。
unreadable images 添付時の crash を text placeholder 化で修正した。
zero-byte や corrupt 画像が paste、MCP、dialog で添付された場合だ。
auto-mode classifier の data exfiltration 検出を改善した。
repository contents の bulk transfers が対象だ。
`rm -rf $HOME` が dangerous path として block される修正した。
`HOME` に trailing slash がある場合も対象だ。
`$TMPDIR` が sandboxed/unsandboxed Bash で異なるディレクトリを参照する問題を修正した。
Stdio MCP server subprocesses に環境変数が追加された。
`CLAUDE_CODE_SESSION_ID` と `CLAUDECODE=1` が渡される。
`claude mcp list`/`get` で unapproved `.mcp.json` servers を `⏸ Pending approval` と表示する。
output piped 時の auto-approve 動作を防止する。
`/remote-control` autocomplete に「Disconnect Remote Control」を表示する。
Remote Control  active 時に表示される。
`/claude-api` skill に Opus 4.8 サポートと移行ガイドを追加した。
`CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE` を deprecated とした。
06/01 に削除予定だ。
Opus 4.6 で fast mode を使う場合は `/model claude-opus-4-6[1m]` と `/fast on` を使用する。
streaming tool execution が常に有効化された。
telemetry 無効時や Bedrock/Vertex/Foundry でも利用可能だ。
Claude in Chrome で接続ブラウザを選択可能になった。
`/chrome` →「Select browser…」または chat 内で選択する。
multiple-choice question prompt の使用頻度を減らした。
モデルが判断可能な場合に質問しなくなった。
`/simplify` が cleanup-only review を実行するようになった。
reuse、simplification、efficiency、altitude をレビューし修正を適用する。
full `/code-review --fix` bug-hunting review は実行しない。
`/effort` slider labels を「Faster」「Smarter」に変更した。
「Speed」「Intelligence」から変更され、 clarity が向上した。

## まとめ

このアップデートで処理能力と拡張性が向上した。
Opus 4.8 による高速化と動的ワークフローで大規模タスクが可能だ。
プラグインシステムの強化でカスタマイズ性が拡大する。
多数の修正により安定した開発環境を提供する。
