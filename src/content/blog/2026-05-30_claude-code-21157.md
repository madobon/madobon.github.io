# Claude Code 2.1.157 リリース：プラグイン自動読み込みと新機能

## はじめに

みなさん、こんにちは！Claude Code が最新バージョン 2.1.157 にアップデートされました。今回の更新では、プラグイン機能の大幅な強化をはじめ、エージェント管理の改善、そして多数のバグ修正が含まれています。

特に 2.1.154 で登場した Opus 4.8 や动态ワークフロー機能と合わせることで、より強力な開発環境が手に入ります。今回は、最新の changelog を元に、主に 2.1.157 の変更点と、直近のバージョンで追加された重要な機能について解説していきます。

## 変更点の概要

今回のアップデートにおける主なハイライトは以下の通りです。

- **プラグイン機能の進化**: `.claude/skills` ディレクトリのプラグインが自動的に読み込まれるようになりました
- **エージェント管理の強化**: `settings.json` の設定反映や、バックグラウンドセッションの操作が改善されました
- **Opus 4.8 の安定化**: 思考ブロックの修正や、高速モードのコストパフォーマンスが向上しています
- **UX の改善**: WSL での画像貼り付けや、ターミナルの描画パフォーマンスなど、日常的な不便さが解消されました

## 詳細解説

### プラグイン機能の自動化と scaffolding

これまでマーケットプレイスからのインストールが必要だったプラグインが、ローカルディレクトリから自動的に読み込まれるようになりました。`.claude/skills` ディレクトリにプラグインを配置するだけで OK です。

また、新しいプラグインを作成するためのコマンドも追加されました。以下のコマンドで簡単にスケルトンを作成できます。

```bash
claude plugin init <name>
```

これにより、カスタムスキルの開発がぐっと身近になります。さらに、`/plugin` コマンド入力時のオートコンプリートも強化され、インストール済みプラグインや既知のマーケットプレイス候補がすぐに表示されるようになりました。

### エージェントとワークツリーの管理改善

`claude agents` の挙動がより柔軟になりました。`settings.json` に記述された `agent` フィールドがディスパッチセッションで尊重されるようになり、必要に応じて `--agent <name>` でオーバーライド可能です。

また、セッション中に Claude 管理のワークツリーを切り替える `EnterWorktree` 機能も強化されました。エージェント終了後にワークツリーがロックされなくなったため、`git worktree remove` や `prune` でのクリーンアップがスムーズに行えます。

バックグラウンドセッションの操作も便利になりました。エージェントビューで `! <command>` と入力するだけで、シェルコマンドをバックグラウンドセッションとして実行し、必要に応じてアタッチ・デタッチできます。

```bash
# コマンドラインからもバックグラウンド実行が可能
claude --bg --exec '<command>'
```

### Opus 4.8 と动态ワークフロー

少し前のバージョン 2.1.154 で登場した Opus 4.8 も、今回のアップデートでさらに安定しました。思考ブロックの修正により API エラーが減少し、デフォルトで「高努力」モードが適用されるようになっています。

 hardest タスクには `/effort xhigh` を使用することで、より深い推論が可能になります。また、动态ワークフロー機能により、Claude にワークフローの作成を依頼すると、バックグラウンドで数十から数百のエージェントをオーケストレーションできるようになりました。

```bash
# ワークフローの実行状況を確認
/workflows
```

### 日々の使いやすさを向上させる修正

地味ですが重要な修正も多数含まれています。

- **WSL 環境**: 画像の貼り付け（`alt+v`）や、Windows エクスプローラーからのドラッグアンドドロップがサポートされました
- **ターミナル描画**: 長い会話や再開時のパフォーマンスが向上し、 redundant なメッセージレンダリングが削減されました
- **クリップボード**: tmux 内でのコピー選択や、VS Code 統合ターミナルでの右クリック貼り付けの重複問題が修正されました
- **サンドボックス**: 起動時の「bash commands will be sandboxed」バナーが削除され、よりクリーンな起動体験になりました

また、`/terminal-setup` コマンドを使用すると、VS Code や Cursor などの統合ターミナルで GPU 加速を無効にし、文字化けを防ぐことができます。

## まとめ

今回の 2.1.157 アップデートは、プラグイン開発のハードルを下げる機能と、既存機能の安定化が両立された素晴らしいリリースです。特に `.claude/skills` による自動読み込みは、自分だけのカスタムコマンドを簡単に共有・管理できるため、生産性向上に大きく貢献するでしょう。

Opus 4.8 の高速モードや动态ワークフロー機能と組み合わせれば、大規模なタスクも効率的にこなせます。ぜひ最新バージョンにアップデートして、新しい機能を試してみてください。
