# Codexと進めたテーマ切替実装とMermaid再描画の調整ログ

今日はこのサイトに、ダークモードとライトモードの切替を入れました。

単にトグルを置くだけでなく、

- 既存のデザインの雰囲気を壊さないこと
- ブログ記事内の `mermaid` 図もテーマに追従すること
- 最後まで commit と push まで流れを切らずに進めること

まで含めて、Codex と対話しながら一気に進めた記録です。

## 最初の依頼

出発点はかなりシンプルで、要件は「ダークモードとライトモードを切り替えできるようにして」でした。

このサイトは Vue ベースで、見た目は `src/style.css` の CSS 変数にかなり寄せてあるので、まずはどこに状態を持たせるのが自然かを見てもらいました。

結果として、

- テーマ状態は `src/App.vue` に持つ
- 配色は `src/style.css` の変数を2テーマ化する
- 保存先は `localStorage`
- 初回表示は OS の `prefers-color-scheme` を尊重する

という構成で進めることになりました。

## 実装でやったこと

Codex は先にリポジトリを見て、共通レイアウト、グローバル CSS、ブログ詳細ページの Mermaid 描画処理を確認してから手を入れていました。

実際に入った変更は大きく3つです。

### 1. ヘッダーにテーマトグルを追加

`src/App.vue` にテーマ状態を追加して、ヘッダーから切り替えられるようにしました。

- 現在テーマを `ref` で保持
- `document.documentElement.dataset.theme` に反映
- `localStorage` に保存
- 初期ロード時のちらつきを減らすため、head に初期化スクリプトも追加

見た目としては、既存のガラスっぽいヘッダーに馴染むようなボタンになっています。

### 2. CSS変数をダーク / ライト両対応に整理

今回の変更で一番大きかったのはここでした。

もともと暗色前提で作っていた配色を、単純な反転ではなく、

- 背景
- テキスト
- カード
- ナビゲーション
- ボタン
- コードブロック

まで一通りテーマ変数化して、ライトモードでもこのサイトらしさが残るように整えました。

明るいテーマは真っ白ベースにはせず、少し紙っぽい温度感のある色に寄せたので、切り替えたときにも雰囲気が急に別物になりすぎないのがよかったです。

### 3. Mermaidを現在テーマに追従させた

ブログ詳細ページでは `mermaid.run()` を後から呼んで図を描画しています。

そのため、テーマ切替に合わせて Mermaid 側も

- ダーク時は `dark`
- ライト時は `default`

で再初期化するようにしました。

ここまでは比較的素直でした。

## 途中で詰まったところ

最初の実装を入れたあと、見た目のテーマ切替自体は動いたのですが、Mermaid の再描画だけが少し怪しい状態でした。

テーマを切り替えても図がうまく更新されず、再描画されていないように見えたので、そこを改めて見直しました。

原因は、`mermaid.run()` が一度走ったあとの `.mermaid` 要素が、すでに SVG を持った状態になっていることでした。

つまり、テーマ変更後にそのまま再実行しても、

- 元の Mermaid ソースが残っていない
- `data-processed` が付いたまま

という条件で、素直には描き直せませんでした。

## 最終的な修正

そこで、再描画前に毎回次の処理を入れました。

1. 元の Mermaid ソースを `data-mermaid-source` に保存する
2. 再描画時にそれを `.textContent` へ戻す
3. `data-processed` を外す
4. そのうえで `mermaid.run()` を呼ぶ

これで、テーマ切替のたびに図をきれいに描き直せるようになりました。

「一回描画された DOM をそのまま使い回せる」と思い込むと詰まりやすいところなので、今回の学びとしてかなり印象に残りました。

## 対話で進めてよかったこと

今回のやり取りで特によかったのは、Codex が最初からファイルを書き始めるのではなく、

- まずどこが責務を持っているか見る
- 実装方針を絞る
- 変更後に `vp check` と `pnpm run build:site` で確認する
- 不具合が見つかったら原因を切り分けて最小修正を入れる

という流れで進めてくれたことでした。

単に「コードを出す」より、ペアで進める感覚に近かったです。

Mermaid の再描画が怪しかったときも、こちらが「うまくいってなさそう」と短く返すだけで、関連箇所を読み直して、原因候補を絞って、修正まで持っていけました。

## 最後にやったこと

修正が落ち着いたあとは、そのまま `git-tools:commit-push` のスキルを使って commit と push まで実施しました。

コミットメッセージは次のとおりです。

```bash
feat: add site theme toggle
```

実装、調整、検証、コミットまで対話の流れを切らずに完結できたので、こういう小さめの改善はかなり相性がよさそうです。

## まとめ

今回の作業は「テーマ切替を入れる」という単発タスクでしたが、実際には

- UI 実装
- グローバル配色整理
- Mermaid の動的描画調整
- 検証
- commit / push

まで含む、そこそこ厚みのあるセッションになりました。

それでも、要所だけ短く指示しながら進めれば、Codex とかなり自然に詰めていける感触がありました。

特に、動いたと思ったあとに出てきた Mermaid の再描画問題まで含めて記録に残しておくと、次に同じような修正をするときにも役立ちそうです。
