# ブログ本文の右余白を追いながら、詳細ヘッダーの引き算までやった

ブログ記事の見た目を見直していた流れで、本文の右側に不自然な余白が残る状態が気になり、細かく追うことになりました。

最初は `blog-prose` や導入段落のスタイルが原因だと思っていたのですが、最終的にはもっと手前の共通スタイルが効いていました。そこから流れで、ブログ詳細ページのヘッダーも「足し算」ではなく「引き算」で整え直すことになりました。

## 最初の依頼

出発点は「デザインが崩れている」という確認でした。

最初に見えていたのは、本文の最初の行とそれ以降で折り返し位置が違って見えることと、記事の右側だけ変に空いて見えることでした。

対象はブログ詳細ページで、実際に触ったのは `src/style.css` の `blog-prose` 周辺です。

## 実装でやったこと

最初は素直に、本文専用のスタイルを順番に見直しました。

まず導入段落だけ別タイポになっていた `.blog-prose > p:first-of-type` を整理して、1 段落目だけ文字サイズや行間が変わらないようにしました。

そのあと `blog-prose` 自体の幅や、段落ごとの `max-width` を調整しながら、「どの指定が右余白を作っているのか」をひとつずつ切り分けていきました。

途中では `.blog-prose` を左寄せにしたり、逆に全幅にしたりも試しましたが、どれも見た目の一部は改善する一方で、別の違和感が残りました。

## 途中で詰まったところ

一番ややこしかったのは、原因が `blog-prose` の中にあるように見えて、実際にはそこだけではなかったことです。

段落ごとの `68ch` 制限を外した時点でも、まだ右側が不自然に空いて見えました。

そこで改めて共通スタイルまで遡って見ると、`src/style.css` には `.card p, .page-heading p` に対して `max-width: 62ch` を入れている箇所があり、これがブログ記事カードの本文段落にもそのまま当たっていました。

つまり「本文専用のスタイルが悪い」というより、カード全体に効かせた共通の読み幅制限が、ブログ詳細ページでは強すぎた、というのが本当の原因でした。

## 最終的な修正

最終的には `blog-prose` 側でこの共通指定を打ち消す形にしました。

`src/style.css` の `.blog-prose p, .blog-prose li` に `max-width: none` を入れて、カード共通の `62ch` 制限を外しています。

そのうえで `.blog-prose` 自体は `max-width: 92ch` と `margin: 0 auto` を残して、本文コンテナとしての適度な読み幅は維持しました。

さらに導入段落だけの強調スタイルも削っているので、最初の行だけ折り返し位置が違って見える状態も一緒に収まりました。

そのあと、空いて見える右側をどう扱うかの流れで、ブログ詳細ページのヘッダー自体にも少し手を入れました。

最初は右余白を埋めるための情報や装飾を足す方向も試したのですが、結果としてはオーバーになりやすく、公開日やタグが上下で重複して見える状態も出ました。

最終的には `src/pages/BlogPostPage.vue` でヘッダー上部の `Published` と `Tags` を消して、情報は本文直前のメタ帯にまとめています。

そのうえで `src/style.css` 側では、`blog-page-heading` の summary 幅やタイトル下の余白を少しずつ調整したものの、途中でタイトルを締めすぎて逆に不自然になりました。

ここは最後に方針を戻して、見た目を足すよりも、タイトルと summary の間隔を `h1` の下マージンで素直に作るだけのシンプルな実装へ寄せています。

## まとめ

今回は最初に `blog-prose` の幅指定ばかり追っていたのですが、実際にはもっと広いスコープの `.card p` が効いていた、というデバッグでした。

コンポーネント専用の見た目を調整しているつもりでも、共通スタイルの影響が残っていると原因の切り分けが難しくなるので、途中で一段引いて見るのは大事だと改めて感じました。

また、今回の後半では「右が空いて見えるから何かを足したい」と思っても、そこで装飾や情報を増やすと、すぐにバランスが崩れることも実感しました。

ブログ詳細ページのような読み物の面では、余白の違和感は新しい要素を足して解決するより、まず既存情報の重複を減らして、間隔を素直に整えるほうがうまくいきやすそうです。

最後に `vp check` と `pnpm run build:site` を通し、本文幅の修正自体は `fix(blog): normalize prose width and intro styling` というコミットで `main` へ push しています。そのあともローカルではヘッダー余白の調整を続けて、どこまで触ると過剰になるかを見ながら整え直しました。
