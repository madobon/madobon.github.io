# Hermes Agent と Claude Code の違いを整理してみた

ターミナル上で自然言語で指示を出して、ファイル編集やコマンド実行、コードレビューまで任せられる AI ツールが増えてきた。今回は、その中でも特に注目されている **Hermes Agent** と **Claude Code** を、実際に使った感触も交えながら比較してみる。

## 一言で言うと

- **Claude Code**: コードベースの理解と編集に特化した「高機能コーディングアシスタント」
- **Hermes Agent**: スキルと記憶を蓄積し、外部ツールや子エージェントと連携する「汎用オーケストレーター」

## 比較表

| 項目                 | Claude Code                  | Hermes Agent                                 |
| -------------------- | ---------------------------- | -------------------------------------------- |
| 主な用途             | コード編集・開発特化         | 開発 + ブラウザ操作 + 自動化                 |
| スキル保存           | 限定的（プロジェクト内設定） | 明示的なスキルファイルとして永続化           |
| 過去セッション検索   | 基本的に現在のセッション内   | `session_search` で全セッション横断検索      |
| サブエージェント委任 | 単体動作                     | `delegate_task` で子エージェント起動・連携   |
| MCP 対応             | 対応している                 | ネイティブ + mcporter CLI で柔軟に統合       |
| ブラウザ操作         | 基本的に非対応               | `browser_navigate`, `browser_click` など対応 |
| cron 管理            | 非対応                       | `cronjob` で定期タスク自動化                 |
| 画像分析             | 非対応                       | `vision_analyze` で AI 画像分析              |
| スマートホーム       | 非対応                       | `openhue` などスキル連携で制御可能           |

## 各項目の詳細

### スコープ・用途の違い

**Claude Code** はリポジトリ内を深く理解し、ファイル間の依存関係を把握してコード変更を提案するのが得意。`git commit` や `gh pr create` まで一気通貫で処理できる。ターミナルに常駐して「コーディングの延長」として使うイメージ。

一方 **Hermes Agent** は開発だけでなく、ブラウザ操作（サイトのスクレイピングやフォーム入力）、cron ジョブ管理、画像分析、スマートホーム制御など、より広範なタスクをカバーする。エージェント自体が「作業の統括者」として振る舞う。

### スキル・メモリシステム

これが大きな違いの一つ。Hermes Agent はタスクが成功すると、手順を「スキル」として `~/.hermes/skills/` にファイルとして保存する。次回同じようなタスクが来たら、そのスキルを自動読み込みして再現性のある動作をする。

また `memory` ツールでユーザー設定や環境情報を永続化できる。セッションが切れても「このユーザーは pnpm を使う」「GitHub の認証方法は SSH」といった情報を保持し続ける。

Claude Code は会話履歴は保持するが、明示的なスキルファイル化や、システム全体に渡る永続メモリの概念は限定的だ。

### クロスセッション検索

Hermes Agent は `session_search` で過去の全セッションを検索できる。「前回の続き」「前にこの話題で何をしたか」を AI が自動的に把握して文脈を復元する。長期プロジェクトで「2週間前に決めた設計」を忘れずに引き継げるのが強み。

### ツールセットの違い

Claude Code のツールは基本的に「コードベース内のファイル操作・コマンド実行」に集中。対して Hermes Agent は以下のようなツールを持つ。

- `browser_navigate`, `browser_click`, `browser_type` ... ブラウザの自動操作
- `cronjob` ... 定期ジョブの作成・管理
- `vision_analyze` ... 画像の AI 分析
- `text_to_speech` ... 音声合成
- `delegate_task` ... 子エージェントへのタスク委任

これらは独立した CLI ツールとしても使えるように設計されており、エージェントが「人間の代わりにブラウザを開いて調べ、結果をコードに反映する」といった動作が可能になる。

### サブエージェント委任

Hermes Agent の `delegate_task` は強力な機能で、Claude Code そのものを子プロセスとして起動して並列処理できる。つまり「Hermes が統括して、Claude Code にコーディングを任せる」という階層構造が組める。

他にも OpenAI Codex、OpenCode など他社のエージェントも子プロセスとして連携可能で、マルチエージェント構成の中核になれる。

### MCP 対応

両方とも MCP（Model Context Protocol）には対応しているが、Hermes Agent は「ネイティブ MCP クライアント」として動作する。`config.yaml` にサーバー設定を書くだけで自動的にツールを発見・呼び出しできる。さらに `mcporter` CLI で ad-hoc にサーバー接続も可能で、統合の柔軟性が高い。

## どちらを選ぶべきか？

**Claude Code を選ぶ場面**

- 大規模なコードベースのリファクタリングや、複数ファイルにまたがる変更を頻繁に行う
- プルリクエスト作成、レビュー、CI 確認などの開発サイクルを閉じたい
- ターミナル内で完結するコーディング作業が中心

**Hermes Agent を選ぶ場面**

- コーディングに加えて、調査（ブラウザ操作）、定期タスク（cron）、ドキュメント作成などが混在する
- 同じ手順を何度も繰り返すので、自動化・スキル化の価値が高い
- マルチエージェント構成で、異なる AI ツールを連携させたい
- セッションを跨いだ記憶・文脈保持が重要

## 実際の組み合わせもあり

実は「Hermes Agent で統括し、コーディング部分は Claude Code に委任する」という使い方も `delegate_task` で可能だ。統合のハブとして Hermes、専門タスクとして Claude Code、という棲み分けも検討できる。

## まとめ

どちらが優れているかではなく、**解決したい問題のレイヤーが違う**。

- Claude Code は「コードを書く・直す」という微細な作業を加速する
- Hermes Agent は「タスク全体を受け持ち、必要に応じてツールや子エージェントを調達する」

個人的には、日常の雑多な作業（調査 + コーディング + 自動化）を一貫して回したい場合は Hermes Agent の汎用性が便利。純粋にコードの品質と速度を追求するなら、Claude Code の深いコードベース理解力が強みになる。
